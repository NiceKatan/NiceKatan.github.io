「 Unique Paths II 」

动态规划 Medium 美区每日一题

  • Problem：A robot is located at the top-left corner of a m x n grid (marked ‘Start’ in the diagram below). The robot can only move either down or right at any point in time. The robot is trying to reach the bottom-right corner of the grid (marked ‘Finish’ in the diagram below).

    Now consider if some obstacles are added to the grids. How many unique paths would there be?

    An obstacle and space is marked as 1 and 0 respectively in the grid.

  • Example：

Input：obstacleGridobstacleGridobstacleGrid = [ [0, 0, 0], [0, 1, 0], [0, 0, 0] ]
Output：2
Explanation：There is one obstacle in the middle of the 3x3 grid above.
There are two ways to reach the bottom-right corner:

  • Right -> Right -> Down -> Down
  • Down -> Down -> Right -> Right

2D DP array

This is a typical 2D dp problem . We define dp[i][j]dp[i][j]dp[i][j] as the number of paths to reach the point [i][j][i][j][i][j] . The robot can only move either down or right at any point in time , so for each point , it can only be reached by the adjacent point to the left or above.

  • When obstacle[i][j]=1obstacle[i][j]=1obstacle[i][j]=1 , grid[i][j]grid[i][j]grid[i][j] has obstacle , all the paths passing this point will no longer valid so the dp[i][j]=0dp[i][j]=0dp[i][j]=0
  • When obstacle[i][j]=0obstacle[i][j]=0obstacle[i][j]=0 , grid[i][j]grid[i][j]grid[i][j] can be reached by grid[i][j−1]grid[i][j-1]grid[i][j−1] and grid[i−1][j]grid[i-1][j]grid[i−1][j] , so the number of arrival paths of the current grid is dp[i][j]=dp[i−1][j]+dp[i][j−1]dp[i][j] = dp[i-1][j]+dp[i][j-1]dp[i][j]=dp[i−1][j]+dp[i][j−1]
