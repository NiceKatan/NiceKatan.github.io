「 Count Binary Substrings 」

字符串 Easy 美区每日一题

  • 题目：给定一个字符串 sss，计算具有相同数量 0 和 1 的非空（连续）子字符串的数量，并且这些子字符串中的所有 0 和所有 1 都是连续的。重复出现的子串要计算它们出现的次数。
  • 示例：

输入：“00110011”
输出：6
解释：有6个子串具有相同数量的连续 1 和 0 : “0011”，“01”，“1100”，“10”，“0011” 和 “01”。请注意，一些重复出现的子串要计算它们出现的次数。

注意："00110011"不是有效的子串，因为所有的0（和1）没有组合在一起。

基本思路

假定给的给的子串为 "000011100010011" ，我们把子串按照连续的 0 和 1 分割开得到：

  • 0000 111 000 1 00 11

统计字串各段连续字符的个数得到数组： [4,3,3,1,2,2][4, 3, 3, 1, 2, 2][4,3,3,1,2,2]，可知数组中每两个相邻的数，代表的是两种不同字符的个数，如 [4,3,3][4, 3, 3][4,3,3] 中 [4,3][4, 3][4,3] 代表有 4个连续的0 和 3个连续的1， [3,3][3, 3][3,3] 代表有 3个连续的1 和 3个连续的0。每一对相邻的连续字符可以组成符合要求的子串数量取决于两者间更小的连续字符的长度，比如 0000 111 可以组成 000111、0011、01三种子串，数目为更短连续子串 111 的长度。

空间复杂度O(n)O(n)O(n)的解法是遍历字符串并统计连续字符的个数存入数组，再遍历数组相邻数对求解，但我们可以进一步优化至O(1)O(1)O(1)的空间复杂度：

  • 采用 curcurcur 变量记录当前连续字符个数，preprepre 变量记录上一段连续字符个数，字符不再连续时，取 min(cur,pre)min(cur, pre)min(cur,pre) 加入答案，并将 preprepre 更新为 curcurcur。

代码

  • 时间复杂度：O(n)O(n)O(n)，遍历字串一次
  • 空间复杂度：O(1)O(1)O(1)，常量字符空间
