「 Find Minimum In Rotated Sorted Array II 」

二分查找 Hard 每日一题

  • 题目：已知一个长度为nnn的数组，预先按照升序排列，经由1到 nnn 次旋转后，得到输入数组。例如，原数组 numsnumsnums =[0,1,4,4,5,6,7]在变化后可能得到：

    • 若旋转 4 次，则可以得到 [4,5,6,7,0,1,4]
    • 若旋转 7 次，则可以得到 [0,1,4,4,5,6,7]

    数组 [a[0],a[1],a[2],...,a[n−1]][a[0], a[1], a[2], ..., a[n-1]][a[0],a[1],a[2],...,a[n−1]] 旋转一次的结果为数组 [a[n−1],a[0],a[1],a[2],...,a[n−2]][a[n-1], a[0], a[1], a[2], ..., a[n-2]][a[n−1],a[0],a[1],a[2],...,a[n−2]]。

    给你一个可能存在重复元素的数组 numsnumsnums，它原来是一个升序排列的数组，并按上述情形进行了多次旋转。请你找出并返回数组中的最小元素。

  • 示例：

输入：numsnumsnums = [2,2,2,0,1]
输出：0

基本思路

我们假定最小值为 pivotpivotpivot，对于已经旋转后的数组，[0:pivot−1][0:pivot-1][0:pivot−1] 内元素一定大于等于 [pivot:N−1][pivot:N-1][pivot:N−1] 内元素，并且 [0:pivot−1][0:pivot-1][0:pivot−1] 与 [pivot:N−1][pivot:N-1][pivot:N−1] 中元素均单调递增，因对此我们可以使用二分查找。

设定 leftleftleft 与 rightrightright 为数组左右边界，对于每一次区间判别，比较 nums[mid]nums[mid]nums[mid] 与 nums[right]nums[right]nums[right] ：

  • nums[mid]<nums[right]nums[mid]<nums[right]nums[mid]<nums[right]：说明 nums[mid]nums[mid]nums[mid] 处于 [piovt:N−1][piovt:N-1][piovt:N−1] 区间中，(mid:right](mid:right](mid:right] 都比最小值大，由于最小值 pivotpivotpivot 也小于 nums[right]nums[right]nums[right]，所以 nums[mid]nums[mid]nums[mid] 可能为最小值，下一轮搜索范围 [left,mid][left,mid][left,mid]，缩小右边界 right=midright=midright=mid
  • nums[mid]>nums[right]nums[mid]>nums[right]nums[mid]>nums[right]：说明 midmidmid 处于 [0:pivot−1][0:pivot-1][0:pivot−1] 区间中，[left:mid][left:mid][left:mid] 都不是最小值，最小值在 midmidmid 右边，下一轮搜索范围 [mid+1:right][mid+1:right][mid+1:right]，缩小左边界 left=mid+1left=mid+1left=mid+1
  • nums[mid]=nums[right]nums[mid]=nums[right]nums[mid]=nums[right]：说明 nums[mid]nums[mid]nums[mid] 与 nums[right]nums[right]nums[right] 重复，不能确定最小值在 midmidmid 的左边还是右边，但由于重复，所以不论 nums[right]nums[right]nums[right] 是不是最小值，都有其替代品 nums[mid]nums[mid]nums[mid]，因此我们可以忽略二分查找的右端点 right=right−1right=right-1right=right−1，参考@LeetCode题解
